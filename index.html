Pourquoi les gens utilisent-ils Python ?
  • Qualité logicielle:

    • Le code Python est conçu pour être lisible, et donc réutilisable et maintenable, bien plus que les langages de script traditionnels

  • Productivité des développeurs:

    • Le code Python est généralement d'un tiers à un cinquième de la taille du code C++ ou Java équivalent. Cela signifie qu'il y a moins à taper, moins à déboguer et moins à maintenir.

  • Portabilité des programmes:

    • La plupart des programmes Python s'exécutent sans modification sur toutes les principales plates-formes informatiques.

  • Bibliothèques de soutien:

    • Python peut être étendu à la fois avec des bibliothèques internes et une vaste collection de logiciels de support d'applications tiers

  • Intégration des composants:

    • Les scripts Python peuvent facilement communiquer avec d'autres parties d'une application, grâce à divers mécanismes d'intégration.

Comments

Les commentaires de type hash-mark (#) sont le moyen le plus élémentaire de documenter votre code. Python ignore simplement tout le texte qui suit un #, Vous pouvez donc faire suivre ce caractère de n'importe quel mot ou description utile aux programmeurs 

                  #This is a comment
                  print("Hello, World!")

Les commentaires peuvent être placés à la fin d'une ligne, et Python ignorera le reste de la ligne

                  print("Hello, World!") #This is a comment

Un commentaire ne doit pas nécessairement être un texte qui explique le code, il peut également être utilisé pour empêcher Python d'exécuter du code

                  #print("Hello, World!")
                  print("Cheers, Mate!")

Liste

Une liste est une collection ordonnée de valeurs. Chaque valeur est appelée un élément, et chaque élément a une position numérique dans le tableala listeu, appelée son index. Les listes Python sont non typés : un élément de tableau peut être de n'importe quel type, et différents éléments d'un même tableau peuvent être de types différents.

Creating Lists

La façon la plus simple de créer une liste est d'utiliser un littéral de tableau, qui est simplement une liste d'éléments de tableau séparés par des virgules entre crochets 

                  empty = [] # Un tableau sans éléments
                  primes = [2, 3, 5, 7, 11] # Un tableau avec 5 éléments numériques
                  misc = [ 1.1, True, "a", ] # 3 éléments de différents types + virgule de fin de ligne

Les valeurs d'un littéral de tableau ne doivent pas nécessairement être des constantes ; elles peuvent être des expressions arbitraires 

                base = 1024
                table = [base, base+1, base+2, base+3]
Lecture et écriture des éléments d'une liste

Vous pouvez utiliser cette syntaxe pour lire et écrire la valeur d'un élément d'une liste. Ainsi, les instructions suivantes sont toutes légales en Python :

Les Sets sont utilisés pour stocker plusieurs éléments dans une seule variable. Un set est une collection qui est non ordonnée, non modifiable et non indexée. Les ensembles ne peuvent pas avoir deux éléments ayant la même valeur

Dictionary

Les dictionnaires sont utilisés pour stocker des valeurs de données dans des paires clé:valeur. Un dictionnaire est une collection qui est ordonnée, modifiable et qui ne permet pas les doublons.

Les dictionnaires sont écrits avec des crochets, et ont des clés et des valeurs 

                    thisdict =	{
                      "brand": "Ford",

Dictionary

En fonctionnement normal, vous créez des dictionnaires avec des littéraux et vous stockez et accédez aux éléments par clé avec indexation 

                    d = {'spam': 2, 'ham': 1, 'eggs': 3} # Make a dictionary

La fonction intégrée len fonctionne également sur les dictionnaires; elle renvoie le nombre d'éléments stockés dans le dictionnaire ou, de manière équivalente, la longueur de sa liste de clés.

Avant d'entrer dans les détails de l'une ou l'autre des déclarations concrètes, je veux commencer à étudier la syntaxe des instructions Python

Toutes les instructions composées Python (instructions dans lesquelles d'autres instructions sont imbriquées) suivent le même schéma général : une ligne d'en-tête terminée par double point, suivie d'un bloc de code imbriqué, généralement indenté sous la ligne d'en-tête, comme ceci : 

                  Ligne d'en-tête:

Le double point est obligatoire, et l'omettre est probablement l'erreur de codage la plus courante pour les nouveaux programmeurs Python

Pourquoi la syntaxe de l'indentation ?

c'est l'une des principales façons dont Python oblige (presque) les programmeurs à produire un code uniforme, régulier et lisible.

if Tests

Ce chapitre présente l'instruction if de Python, qui est la principale instruction utilisée pour sélectionner des actions alternatives en fonction des résultats des tests.

if Tests - Format général

Lorsque l'instruction if s'exécute, Python exécute le bloc de code associé au premier test dont l'évaluation est vraie, ou le bloc else si tous les tests sont faux. La forme générale d'une instruction if ressemble à ceci 

                    if test1: # if test
                        statements1 # bloc associé
                    elif test2: # Optionnel elifs
                        statements2
                    else: # Optionnel else
                        statements3

Exemple de base 

                    if 1:
                        print('true')

if/else Ternary Expression

Parfois, les éléments d'une déclaration sont si simples qu'il semble exagéré de les répartir sur plusieurs lignes.

Python nous offre un format d'expression qui nous permet de faire des if/else dans une seule expression. 

A = Y if X else Z

Navigation exemple

                direction = "Toulouse"
                result = "aller à gauche" if direction == "Toulouse" else "aller à droite"

while Loops

L'instruction while de Python est la construction d'itération la plus générale du langage. En termes simples, elle exécute de manière répétée un bloc d'instructions (normalement indentées) tant qu'un test situé en haut de l'instruction continue d'être évalué à une valeur vraie. On l'appelle une "boucle" parce que le contrôle revient au début de l'instruction jusqu'à ce que le test devienne faux.

Format général

                      while test : # Test en boucle
                          statements # Corps de la boucle
                      else : # Optionnel else
                          déclarations # Exécution si la boucle n'a pas été interrompue par break

Exemple

                      while True :

break, continue, pass, et le Loop else

break: Saute hors de la boucle la plus proche (au-delà de l'énoncé de la boucle).

continue: Saut au sommet de la boucle la plus proche (à la ligne d'en-tête de la boucle).

pass: Ne fait rien du tout : il s'agit d'un espace vide pour une déclaration.

Loop else block: S'exécute si et seulement si la boucle est quittée normalement (c'est-à-dire sans coupure).

                  while test :
                      declarations
                      if test : break # Quitter la boucle maintenant, ignorer else si présent
                      if test : continue # Aller au début de la boucle maintenant, à test1
                  else :
                      statements # Exécuter si nous n'avons pas trouvé de 'break'.

for Loops

La boucle for est un itérateur générique en Python : elle peut parcourir les éléments d'une séquence ordonnée ou d'un autre objet itérable.


Format général

                  for target in object : # Affecter les éléments de l'objet à la cible
                      statements # Corps de la boucle répétée : utilise la cible
                  else : # Partie else facultative
                      statements # Si nous n'avons pas atteint un 'break' (pause)

Lorsque Python exécute une boucle for, il affecte les éléments de l'objet itérable à la cible un par un et exécute le corps de la boucle pour chacun. Le corps de la boucle utilise généralement la cible d'affectation pour faire référence à l'élément actuel de la séquence comme s'il s'agissait d'un curseur qui se déplace dans la séquence.

for Loops - break, continue, pass, et le Loop else

L'instruction for prend également en charge un bloc else facultatif, qui fonctionne exactement comme dans une boucle while - il est exécuté si la boucle sort sans tomber sur une instruction break (c'est-à-dire si tous les éléments de la séquence ont été visités). Les instructions break et continue introduites précédemment fonctionnent également de la même manière dans une boucle for que dans une boucle while. Le format complet de la boucle for peut être décrit de la manière suivante

                    for target in object : # Affecter les éléments de l'objet à la cible
                        declarations
                        if test : break # Quitter la boucle maintenant, passer à autre chose
                        if test : continue # Aller au début de la boucle maintenant
                    else :
                        statements # Si nous n'avons pas trouvé de 'break'.

Functions

En termes simples, une fonction est un outil qui regroupe un ensemble d'instructions de manière à ce qu'elles puissent être exécutées plusieurs fois dans un programme - une procédure packagée invoquée par son nom.

  • Une fonction est un bloc de code qui ne s'exécute que lorsqu'il est appelé.
  • Vous pouvez transmettre des données, appelées paramètres, à une fonction.
  • Une fonction peut renvoyer des données comme résultat.
Functions - Pourquoi utiliser des fonctions ?

Avant d'entrer dans les détails, il faut établir une image claire de ce que sont les fonctions

  • Maximiser la réutilisation du code et minimiser la redondance

    • Les fonctions Python constituent le moyen le plus simple de d'emballer la logique que vous souhaitez utiliser à plusieurs endroits et à plusieurs reprises

  • Procedural decomposition

    • Les fonctions constituent également un outil permettant de diviser les systèmes en parties ayant des rôles bien définis. Par exemple, pour faire une pizza à partir de rien, il faut commencer par mélanger la pâte, l'étaler, ajouter les garnitures, la faire cuire, etc. Si vous deviez programmer un robot pizzaïolo, les fonctions vous aideraient à diviser la tâche globale de "fabrication d'une pizza" en plusieurs parties, une fonction pour chaque sous-tâche du processus. Il est plus facile d'implémenter les petites tâches isolément que d'implémenter l'ensemble du processus en une seule fois.

Functions - def Statements ?

L'instruction def crée un objet fonction et l'affecte à un nom. Son format général est le suivant :

                  def name(arg1, arg2,... argN):
                  statements

def se compose d'une ligne d'en-tête suivie d'un bloc d'instructions, généralement indenté. Le bloc d'instructions devient le corps de la fonction, c'est-à-dire le code que Python exécute chaque fois que la fonction est appelée par la suite.


L'instruction Python return peut apparaître n'importe où dans le corps d'une fonction ; lorsqu'elle est atteinte, elle met fin à l'appel de la fonction et renvoie un résultat à l'appelant

Functions - Simple
Functions - Calls/Appelle

L'instruction def crée une fonction mais ne l'appelle pas. Après l'exécution de l'instruction def, vous pouvez appeler (exécuter) la fonction dans votre programme en ajoutant des parenthèses après le nom de la fonction 

                  def times(x, y): # Créer et attribuer une fonction
                    return x * y

Les parenthèses peuvent éventuellement contenir un ou plusieurs arguments d'objet, qui seront passés (assignés) aux noms de l'en-tête de la fonction 

                  times(2, 4) # Arguments in parentheses
                  #La fonction renvoie 8

Maintenant, regardez ce qui se passe quand la fonction est appelée une autre fois, avec des objets très différents types:

                  times('Ni', 4) # Les fonctions sont "sans typologie"."
                  #La fonction renvoie 'NiNiNiNi

Functions - Scopes

Les scopes permettent d'éviter les conflits de noms dans le code de votre programme : les noms définis dans une unité de programme n'interfèrent pas avec les noms d'une autre unité.


Lorsque vous utilisez un nom dans un programme, Python crée, modifie ou recherche le nom dans ce que l'on appelle un espace de noms, c'est-à-dire un endroit où vivent les noms.

L'endroit où vous attribuez un nom dans votre code source détermine l'espace de nom dans lequel il va vivre et donc sa visibilité.

Functions - Scopes

les variables peuvent être assignées à trois endroits différents, correspondant à trois scopes différentes

  • Si une variable est assignée à l'intérieur d'une def, elle est locale à cette fonction.
  • Si une variable est assignée dans une def englobante, elle est non locale aux fonctions imbriquées.
  • Si une variable est assignée en dehors de toutes les defs, elle est globale à l'ensemble du fichier.

Par exemple, dans le fichier de module suivant, l'affectation X = 99 crée une variable globale nommée X (visible partout dans ce fichier), mais l'affectation X = 88 crée une variable locale X (visible uniquement dans l'instruction def) : 

                  X = 99 # Global (module) scope X

                  def func():
                      X = 88 # Local (function) scope X: une autre variable
                      print(X)

                  func() #il affiche 88
                  print(X)  #il affiche 99
Functions - Scopes (LEGB lookup rule)

Python recherche jusqu'à quatre scopes - le local (L) scope, puis les Eclosing locals scopes (E), puis le scope global (G), et enfin la built-in (B) scope - et s'arrête au premier endroit où le nom est trouvé. Si le nom n'est pas trouvé au cours de cette recherche, Python signale une erreur.

Ignorons pour l'instant les built-ins

Functions - La déclaration globale

La déclaration global et sa cousine nonlocal sont les seules choses qui ressemblent de près ou de loin à des déclarations en Python. Ce ne sont pas des déclarations de type ou de taille, cependant ; ce sont des déclarations d'espace de noms. La déclaration global indique à Python qu'une fonction prévoit de de modifier un ou plusieurs noms globaux, c'est-à-dire des noms qui vivent dans namespace (espace de noms) du module qui l'entoure 

                  X = 88 # Global X
                  print(X) # il affiche 88

                  def func():
                      global X
                      X = 99 # Global X: en dehors de def

                  func()
                  print(X) # il affiche 99
Functions - Nested Scope Examples

le def est simplement une instruction exécutable, qui peut apparaître partout où une autre instruction peut apparaître , y compris imbriquée dans un autre def. Ici, le def imbriqué s'exécute pendant qu'un appel à la fonction f1 s'exécute ; il génère une fonction et l'assigne au nom f2, une variable locale dans le scope local de f1. En un sens, f2 est une fonction temporaire qui ne vit que pendant l'exécution de (et n'est visible que par le code de) la fonction f1 qui l'entoure 

                  X = 99 # Global scope name: pas utilisé
                  def f1():
                      X = 88 # Enclosing def local
                      def f2():
                          print(X) # Reference fait dans nested def
                          f2()
                  f1() # affiche 88: enclosing def local
Functions - nonlocal

Le mot-clé nonlocal est utilisé pour travailler avec des variables à l'intérieur de fonctions imbriquées, où la variable ne doit pas appartenir à la fonction interne.

Utilisez le mot clé nonlocal pour déclarer que la variable n'est pas locale.

                      def myfunc1():
                          box = "inception"
                          def leonardoFunction():
                              nonlocal box
                              box = "Leonardo DiCaprio"
                          leonardoFunction()
                          return box
    
                      print(myfunc1()) #affiche Leonardo DiCaprio

Functions - Arguments

Les diapositives suivantes poursuivent l'histoire des fonctions en étudiant les concepts du passage d'arguments en Python, c'est-à-dire la façon dont les objets sont envoyés aux fonctions en tant que données d'entrée.

Comme nous le verrons, les arguments (aussi appelés paramètres) sont affectés à des noms dans une fonction, mais ils ont plus à voir avec les références d'objets qu'avec les scopes de variables.

Functions - Argument-Passing Basics

Les arguments sont transmis en assignant automatiquement des objets aux noms de variables locales.

Les arguments de fonction (références à des objets partagés envoyées par l'appelant) ne sont qu'un autre exemple d'affectation Python à l'œuvre. Les objets passés en tant qu'arguments ne sont jamais copiés automatiquement.

C'est un peu comme faire : box = box2 = "text"

Functions - Argument-Passing Basics

L'affectation des noms d'arguments à l'intérieur d'une fonction n'affecte pas l'appelant.

Les noms des arguments dans l'en-tête de la fonction deviennent de nouveaux noms locaux lorsque la fonction s'exécute, dans la portée de la fonction. Il n'y a pas d'alias entre les noms d'arguments de la fonction et les noms de variables dans le scope de l'appelant.

C'est un peu comme faire : 

                      box = box2 = "text"
                      box2 = "nouveau texte"
Functions - Argument-Passing Basics

La modification de l'argument d'un objet mutable dans une fonction peut avoir un impact sur l'appelant.

D'un autre côté, comme les arguments sont simplement affectés à des objets transmis, les fonctions peuvent modifier les objets mutables transmis sur place, et les résultats peuvent affecter l'appelant. Les arguments mutables peuvent être des entrées et des sorties pour les fonctions

Exemple : 

                      arr = [1,2,3,4]
                      def mutate(arr):
                          arr[0] = 100

                      mutate(arr)
                      print(arr[0]) #100
                      arr = [1,2,3,4]
                      def reassign(arr):
                          arr = [100,200,300,400]

                      reassign(arr)
                      print(arr[0]) #1
Functions - Nombre d'arguments

Par défaut, une fonction doit être appelée avec le nombre correct d'arguments. Cela signifie que si votre fonction attend 2 arguments, vous devez appeler la fonction avec 2 arguments, ni plus, ni moins.

Cette fonction attend 2 arguments, et reçoit 2 arguments :

                def my_function(fname, lname):
                    print(fname + " " + lname)

                my_function("Emil", "Refsnes")

Cette fonction attend 2 arguments, mais n'en reçoit qu'un seul:

                  def my_function(fname, lname):
                      print(fname + " " + lname)

                  my_function("Emil")
Functions - Arguments arbitraires, *args

Si vous ne savez pas combien d'arguments seront passés dans votre fonction, ajoutez un * devant le nom du paramètre dans la définition de la fonction.

De cette façon, la fonction recevra un tuple d'arguments et pourra accéder aux éléments en conséquence:

                  def my_function(*kids):
                      print("Le plus jeune enfant est " + kids[2])

                  my_function("Emil", "Tobias", "Linus")
Functions - Arguments de mots-clés

Vous pouvez également envoyer des arguments avec la syntaxe clé = valeur

De cette façon, l'ordre des arguments n'a pas d'importance:

                      def my_function(child3, child2, child1):
                  print("Le plus jeune enfant est " + child3)

                  my_function(child1 = "Emil", child2 = "Tobias", child3 = "Linus")

Si vous ne savez pas combien d'arguments seront passés dans votre fonction, ajoutez deux astérisques : ** devant le nom du paramètre dans la définition de la fonction.

De cette façon, la fonction recevra un dictionnaire d'arguments, et pourra accéder aux éléments en conséquence: 

                  def my_function(**kid):
                      print("Le plus jeune enfant est " + kid["lname"])

                  my_function(fname = "Tobias", lname = "Refsnes")

Functions - Indications de type

En Python, votre code n'est pas censé se soucier de types de données spécifiques. S'il le fait, il sera limité à travailler sur les types que vous avez prévus lorsque vous l'avez écrit, et il ne supportera pas d'autres types d'objets compatibles qui pourraient être codés dans le futur. Bien qu'il soit possible de tester les types à l'aide d'outils tels que la fonction intégrée de type if isinstance(o, str), la flexibilité de votre code s'en trouve réduite. En général, nous codons en fonction des interfaces d'objets en Python, et non des types de données 

                  def times(x: int, y: int) -> int: # Indications de type
                  return x * y

                  print(times(2,5))
                  print(times("Ha", 5))
                  #tous les appels fonctionnent sans erreur, même si nous avons ajouté une indication de type.

Comme vous commencez tout juste à apprendre Python, vous pouvez attendre d'avoir plus d'expérience avec les indications de type.
